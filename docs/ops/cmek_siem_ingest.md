# CMEK SIEM Ingest Runbook (KS.2.7)

Tier 2 customers ingest their own KMS audit stream. OmniSight provides
the tag / label set and filter contract; the customer's CloudTrail or
Cloud Audit Logs pipeline remains customer-owned.

## 1. Generate the Spec

Tenant admins can request the provider-specific ingest contract:

```http
POST /api/v1/tenants/t-acme/cmek/siem/ingest-spec
Content-Type: application/json

{
  "provider": "aws-kms",
  "key_id": "arn:aws:kms:us-east-1:111122223333:key/00000000-0000-0000-0000-000000000000",
  "principal": "arn:aws:iam::444455556666:role/OmniSightCMEKAccess"
}
```

The response includes `omnisight_tags`, `ingest_filter`,
`required_events`, `customer_ingest_targets`, and stable
`ingest_spec_json` for change-control tickets.

## 2. AWS CloudTrail

Apply these tags to the customer-managed KMS key:

```json
{
  "OmniSight": "true",
  "OmniSightTenantId": "t-acme",
  "OmniSightControl": "cmek",
  "OmniSightSchema": "ks.2.7"
}
```

The customer SIEM should ingest CloudTrail KMS events where:

```text
eventSource = 'kms.amazonaws.com'
AND eventName IN ('Encrypt','Decrypt','DescribeKey')
AND resources.ARN = '<customer KMS key ARN>'
```

If the customer uses the IAM role generated by the CMEK wizard, also
filter on `userIdentity.sessionContext.sessionIssuer.arn`.

CloudTrail events should show OmniSight's encryption context on KMS
`Encrypt` / `Decrypt`:

```json
{
  "tenant_id": "t-acme",
  "schema": "ks.1.2",
  "purpose": "cmek-tenant-dek"
}
```

Supported customer ingest targets are CloudTrail Lake, S3 with a SIEM
forwarder, or an EventBridge partner destination.

## 3. Google Cloud Audit Logs

Apply these labels to the customer-managed CryptoKey:

```json
{
  "omnisight": "true",
  "omnisight_tenant_id": "t-acme",
  "omnisight_control": "cmek",
  "omnisight_schema": "ks-2-7"
}
```

The customer Log Router sink should use a filter like:

```text
resource.type="cloudkms_cryptokey"
protoPayload.serviceName="cloudkms.googleapis.com"
protoPayload.resourceName="<customer CryptoKey resource id>"
```

If the customer granted OmniSight's service account directly, also
filter on `protoPayload.authenticationInfo.principalEmail`.

Supported customer ingest targets are a Log Router sink, Pub/Sub with a
SIEM forwarder, or a BigQuery linked dataset.

## 4. Verification

After enabling the sink, ask the customer to run the CMEK wizard verify
step or another low-risk tenant operation. The SIEM should receive one
or more provider events for `Encrypt`, `Decrypt`, and the provider's
key metadata read (`DescribeKey` on AWS, `GetCryptoKey` on GCP).

Vault Transit is intentionally out of scope for this cloud-native
runbook. Vault audit devices are customer deployment-specific and use
the customer's Vault path / namespace policy rather than CloudTrail or
Cloud Audit Logs.
