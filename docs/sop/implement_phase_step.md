# 任務：執行標準流程

## 執行規則
- 嚴格按照順序執行，不可以跳步驟。

## 步驟清單
### Step 1: 深度分析及評估
- 執行深度分析，並評估該 Phase 的實作風險、衝擊、可能產生的副作用。

### Step 2: 拆分工作
- 將 Phase 的工作拆分成數個子階段。
- 同時評估每個子階段實作的困難度、影響範圍、可能產生的副作用。

### Step 3: 執行子階段實作(每個子階段重複執行)
- 依照規劃實作子階段。
- 完成後確認可正常執行。
- 深度分析及檢查實作後造成的影響及產生的錯誤和副作用。
- 修正實作後產生的問題。
- 確認新問題全數修正完成。
- 確認可以正常執行。
- 清理工作路徑。
- 更新.gitignore。
- 執行git commit。

### Step 4: 重新評估實作後的問題
- 深度分析及檢查此 Phase 實作完成後所造成的問題、影響、錯誤、缺失、和副作用。
- 並逐一進行修正。
- 每修正完一個問題就進行一次檢查。
- 確認該問題修復後可以正常執行。
- 清理工作路徑。
- 更新.gitignore。
- 執行git commit。

### Step 5: 狀態更新
- 進行常態更新。
- 更新HANDOFF.md。
- 更新.README.md。
- 執行git commit。
- 列出該階段的執行總結。

### Step 6: 實際啟動
- 啟動程式展示結果
