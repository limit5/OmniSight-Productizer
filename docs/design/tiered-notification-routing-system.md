分級通知路由系統設計(Tiered Notification Routing System)

本文件定義了 OmniSight-Productizer 系統中的通知與告警架構 (Tiered Notification Routing System)。目的在於為 AI Agent 協同開發過程提供精準的狀態回報，在「不干擾開發者日常心流」與「確保關鍵阻斷事件不被漏接」之間取得完美平衡。

---

## 一、 通知管道與工具定位 (Notification Channels)

系統將根據事件的嚴重程度與急迫性，智慧路由至最適合的外部工具：

| 工具類別 | 代表軟體範例 | 系統定位 (Best Use Case) | 優點與特性 |
| :--- | :--- | :--- | :--- |
| **1. 系統內建 UI** | Dashboard 鈴鐺、站內信 | **歷史追蹤與全域狀態**。收攏所有層級的通知。 | 不打斷工作，使用者登入時即可一目了然。 |
| **2. 即時通訊 (IM)** | Slack, Microsoft Teams, Discord, Google Chat, Telegram, LINE Notify | **日常協作與動態推播**。處理中等優先級的 Code Review 請求。 | 即時性高，支援按鈕卡片可一鍵跳轉至 Gerrit。 |
| **3. 工單系統 (Issue)**| Jira, GitLab Issues | **任務指派與長效追蹤**。處理 AI 無法解決的 Bug 或死迴圈。 | 具備優先級排序，確保技術債或需人類介入的任務不會遺失。 |
| **4. 緊急告警** | PagerDuty, SMS 簡訊 | **系統級災難防禦**。僅在基礎設施斷線或崩潰時觸發。 | 具備值班升級機制 (On-call)，確保重大異常立刻被處理。 |

---

## 二、 四級通知定義與 UI 路由規則 (4-Tier Notification Rules)

系統通知分為四個等級 (L1 ~ L4)，每個等級皆有對應的觸發情境、前端 UI 表現及外部發送規則。

### 🟢 Level 1: Info (資訊與動態)
* **定義：** 系統正常運作的背景事件，僅作紀錄，無需人類介入。
* **觸發情境：** * AI Agent 開始建立工作區並執行任務。
  * Agent 順利產出 `HANDOFF.md` 進行狀態交接。
  * 代碼 Merge 後成功透過 Replication 同步至遠端 Repo。
* **前端 UI：** 靜默紀錄。通知小鈴鐺 **不會** 亮起紅點，僅在使用者點開列表時，以灰階或淡藍色顯示於歷史紀錄中。
* **外部路由：** **無**。(或僅發送至 IM 軟體的 `#system-logs` 靜音頻道)。

### 🟡 Level 2: Warning (警告與待辦)
* **定義：** 任務已到達一個節點，或出現輕微異常，需要人類在「有空時」留意。
* **觸發情境：** * AI 開發者成功在 Gerrit 上建立新的 Patch Set，等待審查。
  * AI 審查員發現潛在問題，給予 `Code-Review -1` 分。
* **前端 UI：** 溫和提示。通知小鈴鐺亮起數字，清單內的通知卡片帶有 **黃色 / 橘色** 標記。
* **外部路由：** 觸發 **IM 即時通訊**，在專屬頻道 (如 `#code-review`) 發送帶有 Gerrit 連結的推播。

### 🟠 Level 3: Action Required (阻斷與請求介入)
* **定義：** 自動化流程卡關，沒有人類的指令，系統與 Agent 無法繼續前進。
* **觸發情境：** * Patch Set 已獲得 AI `+1` 認可，**等待人類主管給予 `+2` 並執行 Merge**。
  * AI 修改程式碼失敗，觸發「重試次數上限」陷入無限迴圈，需人類除錯。
* **前端 UI：** 強烈提示。登入 Dashboard 首頁時，頂部出現顯眼的紅色 Banner，或在工作區呈現獨立的 **紅色待辦卡片 (Task Card)**。
* **外部路由：** 1. 觸發 **IM 即時通訊**，並 Tag 特定的 `@審查者` 或 `@Maintainer`。
  2. 透過 API 自動在 **工單系統 (Jira/GitLab)** 建立一張 Ticket 並指派給負責人。

### 🔴 Level 4: Critical (嚴重系統故障)
* **定義：** 基礎設施或核心服務中斷，導致整個自動化開發系統停擺。
* **觸發情境：** * Gerrit 伺服器無回應 (HTTP 500 / Timeout)。
  * GitLab API Token 過期導致程式碼鏡像備份徹底失敗。
  * Docker 虛擬環境資源耗盡。
* **前端 UI：** 霸屏提示。全站頂部顯示常駐且不可關閉的警報條，或彈出全螢幕 Modal 視窗警告。
* **外部路由：** 1. 觸發 **IM 即時通訊** 的最高層級群組廣播 (@channel / @here)。
  2. 同步觸發 **緊急告警系統 (PagerDuty/SMS)**，撥打電話或發送簡訊給當日值班工程師。
