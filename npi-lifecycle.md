# OmniSight-Productizer: 軟硬整合產品 NPI 全生命週期專案規劃書 (含代工與 OBM 模式)

本文件定義了帶有硬體設備與複雜演算法整合之專案的標準開發生命週期（NPI Lifecycle）。本規劃涵蓋了傳統代工模式（ODM/OEM/JDM）的硬體研發流程，並深度整合了自有品牌（OBM）所需的工業設計（ID/UX）與市場進入（Go-To-Market）平行軌道，可直接作為 Jira Epic / Milestone 的建立基準。

---

## 一、 八階段全生命週期定義與跨部門任務 (The 8-Phase NPI Process)

在 OBM 自有品牌模式下，產品開發不再僅是工程部門的責任，每個階段都必須包含「研發工程」、「設計體驗」與「市場行銷」三條軌道的同步推進。

### Phase 0: 概念成型與規格定義 (Kick-off & Concept)
* **核心目標：** 確立商業目標、終端售價區間、目標受眾與核心技術可行性。
* **階段任務：**
  * **[代工/工程]** 產出產品需求文件 (PRD)、確立目標 BOM Cost 上限。
  * **[OBM 軌道]** 產出市場需求文件 (MRD)、完成競品分析。工業設計師 (ID) 開始繪製外觀草圖。
* **JIRA 任務範例：**
  * `[MKT]` 完成競品價格與規格矩陣分析報告
  * `[ID]` 產出 3 款產品外觀概念圖 (Concept Rendering) 供內部投票
  * `[Biz]` 完成 NRE (研發費用) 報價與硬體開發合約簽署

### Phase 1: EIV (Early Investigation - 早期技術與選型評估)
* **核心目標：** 紙上談兵與關鍵零組件定案，確認技術與成本是否達標。
* **階段任務：**
  * **[代工/工程]** 評估主晶片 (SoC)、感光元件、記憶體；分析演算法效能與零組件交期。
  * **[OBM 軌道]** ID 產出高仿真外觀模型 (Mockup)，UX 團隊開始規劃軟體介面。
* **JIRA 任務範例：**
  * `[HW]` 產出 CPU 算力評估與 Initial BOM 表
  * `[UX]` 完成手機端配套 App 的低保真線框圖 (Wireframe)

### Phase 2: POC (Proof of Concept - 概念驗證)
* **核心目標：** 以最快速度證明核心技術路線可行，並驗證外觀設計與硬體空間的衝突。
* **階段任務：**
  * **[代工/工程]** 使用公版 EVB 搭配現成模組，跑通「影像串流 -> 演算法 -> 輸出」主線程。
  * **[OBM 軌道]** 評估硬體零件（如鏡頭、散熱片）是否能塞進 ID 設計的極致機身中，並進行妥協與修改。
* **JIRA 任務範例：**
  * `[SW]` 在 Fullhan EVB 上跑通基礎影像推流 (UVC) 且 CPU 佔用率低於 40%
  * `[EE/ID]` 評估散熱方案是否符合 ID 的無風扇薄型設計要求

### Phase 3: HVT (Hardware Verification Test - 硬體架構驗證)
* **核心目標：** 點亮第一版自製電路板 (A-Sample / Proto)，確保底層硬體無致命缺陷。
* **階段任務：**
  * **[代工/工程]** 完成 PCB Layout 打樣。量測電源樹、高速訊號完整性，確保 Bootloader 正常運作。
* **JIRA 任務範例：**
  * `[HW]` 完成 PCBA 回板點亮 (Bring-up) 並解決 UVC 訊號衰減問題

### Phase 4: EVT (Engineering Verification Test - 工程驗證測試)
* **核心目標：** 軟硬體與機構的首次完整結合 (B-Sample)，集中火力除錯與內部體驗驗證。
* **階段任務：**
  * **[代工/工程]** 解決機構干涉、軟體 Memory Leak、優化散熱與鏡頭 AE 策略。
  * **[OBM 軌道]** 啟動內部 "Dogfooding" (吃狗糧測試)，收集跨部門使用體驗；行銷團隊啟動包裝設計。
* **JIRA 任務範例：**
  * `[SW]` 修復連續掃描 24 小時後發生的系統 Crash
  * `[QA]` 舉辦內部 Alpha 試用大會，收集 50 份 UX 體驗回饋
  * `[MKT]` 啟動產品包裝盒 (Packaging) 與快速入門指南設計

### Phase 5: DVT (Design Verification Test - 設計驗證測試)
* **核心目標：** 設計凍結 (Design Freeze)，確保可靠度、通過安規，並準備對外宣傳素材。
* **階段任務：**
  * **[代工/工程]** 開立正式模具 (C-Sample)。執行可靠度測試 (落摔/高低溫) 與申請安規 (CE/FCC)。
  * **[OBM 軌道]** 啟動 Beta Testing 封閉測試；行銷團隊拿樣機拍攝宣傳影片與官網素材。
* **JIRA 任務範例：**
  * `[Cert]` 取得 FCC / CE / RoHS 認證報告
  * `[MKT]` 完成產品宣傳影片拍攝與宣傳照後製
  * `[UX]` 根據 Beta 測試結果，大幅優化設備配網連接流程

### Phase 6: PVT (Production Verification Test - 生產驗證測試)
* **核心目標：** 驗證工廠量產良率，並完成市場鋪貨準備。
* **階段任務：**
  * **[代工/工程]** 工廠小批量試產 (50~200台)，驗證 SOP 與自動化測試治具，確保良率。
  * **[OBM 軌道]** 建立初始庫存 (Initial Stock)，業務與通路商確認上架排程，官網進行壓力測試。
* **JIRA 任務範例：**
  * `[MFG]` 完成產線自動化測試程式 (MFC)，試產良率達 95%
  * `[Sales]` 確認首批 5,000 台量產機的海運/空運物流排程
  * `[MKT]` 官方網站 Landing Page 製作完成並上線備戰

### Phase 7: MP & Launch (量產與產品發布)
* **核心目標：** 穩定出貨、引爆市場聲量與產品生命週期維護。
* **階段任務：**
  * **[代工/工程]** 正式大批量生產。準備啟動 Cost Down 替代料驗證。
  * **[OBM 軌道]** 舉辦發布會、解禁媒體評測 (Embargo lift)。研發緊盯首週客訴，準備 OTA 緊急更新。
* **JIRA 任務範例：**
  * `[PR]` 發布產品新聞稿並監控首發日市場輿情
  * `[SW]` 準備 Launch Day (Day 1) 的韌體緊急修復包 (OTA Update)
  * `[Support]` 啟動 Issue Tracking 機制以處理 RMA 售後退換貨

## 二、 四大商業模式對專案排程與管理的影響

在繪製甘特圖與規劃里程碑時，必須根據公司的商業角色大幅調整資源比重：

| 商業模式 | 核心焦點 | 時程與風險特徵 | JIRA / 專案管理重點 |
| :--- | :--- | :--- | :--- |
| **ODM<br>(委託設計)** | 滿足客戶規格，控制研發成本。 | **EIV/POC 極長。** 需要投入大量時間摸索技術。DVT 階段測試壓力大。 | 嚴格控管 EIV/POC 的 NRE 預算停損點，避免侵蝕量產利潤。 |
| **OEM<br>(委託製造)** | 可製造性設計 (DFM)、穩定生產良率。 | **省略 EIV/POC。** 拿到圖紙直接進入 HVT/EVT，且 **PVT 極長**。 | 著重產線自動化測試軟體的開發與組裝 SOP 防呆機制。 |
| **JDM<br>(聯合設計)** | 雙方系統的 API/SDK 完美對接。 | **EVT/DVT 最長且易失控。** 常發生軟硬體整合的互相等待 (Ping-pong)。 | 必須在 HVT 與 EVT 階段設定極度明確的**「雙方移交里程碑 (Handoff)」**。 |
| **OBM<br>(自有品牌)** | 創造市場需求、使用者體驗、準時上市。 | **三軌並行，風險最高。** ID 與行銷會干預工程進度，上市日期為死線 (Deadlines)。 | 建立**跨職能 Epic**。嚴格把控各階段的「軟硬體與行銷對齊點 (Alignment Checkpoints)」，避免產品做完卻錯過銷售旺季。 |
