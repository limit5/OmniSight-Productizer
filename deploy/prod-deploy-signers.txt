# OmniSight production deploy GPG signers (FX.7.9)
#
# Used by scripts/check_deploy_ref.sh + scripts/deploy-prod.sh to gate
# which GPG keys may sign a deploy-eligible git tag or branch tip.
# Intent: prevent a compromised forge account from pushing an
# unsigned (or attacker-signed) ref and having it deployed.
#
# Format: one trusted full GPG fingerprint per line (40 hex chars,
# uppercase, no spaces). Lines starting with `#` and blank lines are
# ignored. Use `gpg --list-keys --fingerprint --with-colons` and copy
# the `fpr` field's 10th column (40-char form, no spaces). Both upper-
# and lower-case hex are accepted; the verifier normalises to upper.
#
# Adding / removing a fingerprint is a deliberate change reviewed in
# a PR — that is the audit trail for "who is allowed to authorise a
# production deploy by signing the tag / branch tip".
#
# NOTE: this file is intentionally committed (fingerprints are not
# secret). The matching public keys must also be imported into the
# prod host's gpg keyring (`gpg --import release-signers.asc`) for
# the verifier to accept a signature — listing a fingerprint here
# without importing the public key fails closed (correct outcome).
#
# === RELEASE SIGNERS ===

# `deploy/release-signers.asc` and must be imported into the prod
# host's gpg keyring before the verifier can accept a signature:
#
